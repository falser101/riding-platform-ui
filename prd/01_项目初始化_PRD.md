# PRD - 项目初始化

## 1. 需求概述

| 字段 | 内容 |
|------|------|
| 需求名称 | 项目初始化 |
| 需求编号 | REQ-001 |
| 需求类型 | 基础设施 |
| 优先级 | 高 |
| 状态 | 已分析 |

### 1.1 需求背景

骑行活动平台需要搭建完整的开发框架，为后续功能开发提供基础设施。项目包含两个主要部分：
- **后台管理系统**：Vue3 + Element UI，用于平台运营管理
- **H5前端**：模拟小程序形态，面向C端用户

### 1.2 目标与范围

**目标**：建立可运行的前后端项目骨架，包含用户认证基础架构

**范围**：
- 项目工程化结构搭建
- UI组件库集成
- 路由配置
- 状态管理
- HTTP请求封装
- 用户认证模块（登录/注册/Token管理）

---

## 2. 功能详细说明

### 2.1 后台管理系统功能点

#### 2.1.1 项目脚手架

| 功能点 | 描述 | 验收标准 |
|--------|------|----------|
| Vite + Vue3 工程 | 使用 Vite 构建工具创建 Vue3 项目 | `npm run dev` 正常启动，端口5173 |
| Element Plus 集成 | 引入 Element Plus 组件库 | 组件可正常渲染，无样式冲突 |
| 项目目录规范 | 遵循 Vue 最佳实践目录结构 | 存在 components/ views/ utils/ api/ assets/ 目录 |
| TypeScript 支持 | 启用 TypeScript | tsconfig.json 正确配置，.vue 文件支持 lang="ts" |

#### 2.1.2 路由系统

| 功能点 | 描述 | 验收标准 |
|--------|------|----------|
| Vue Router 4 集成 | 引入 Vue Router | router/index.ts 正确配置 |
| 基础路由 | 登录页、首页、Layout框架 | 路由切换无白屏 |
| 路由守卫 | 权限控制 | 未登录自动跳转登录页 |

#### 2.1.3 状态管理

| 功能点 | 描述 | 验收标准 |
|--------|------|----------|
| Pinia 集成 | 引入 Pinia 状态管理 | stores/ 目录存在 |
| 用户状态 | 管理登录用户信息 | 刷新页面状态保持 |
| Token 持久化 | 使用 localStorage 存储 | 关闭浏览器后重新打开仍保持登录状态 |

#### 2.1.4 HTTP 请求封装

| 功能点 | 描述 | 验收标准 |
|--------|------|----------|
| Axios 封装 | 统一 HTTP 请求工具 | api/ 目录存在 request.ts |
| 拦截器 | 请求/响应拦截 | Token 自动注入，错误统一处理 |
| 环境变量 | dev/prod 环境配置 | .env.development 和 .env.production 存在 |

#### 2.1.5 布局框架

| 功能点 | 描述 | 验收标准 |
|--------|------|----------|
| Admin Layout | 后台管理整体布局 | 包含 Header/Sidebar/Content 区域 |
| 响应式 | 侧边栏折叠 | 窗口缩小时侧边栏可折叠 |

## 3. 用户故事

### 3.1 后台管理系统

| 用户故事 | 场景描述 | 验收要点 |
|----------|----------|----------|
| 作为新入职的运营人员 | 我首次登录后台系统 | 需要能通过手机号验证码登录，看到管理后台主界面 |
| 作为管理员 | 我需要管理多个页面 | 需要清晰的侧边栏导航和顶部操作区 |
| 作为开发者 | 我需要维护项目代码 | 需要规范的目录结构，容易定位代码 |

### 3.2 H5 前端

| 用户故事 | 场景描述 | 验收要点 |
|----------|----------|----------|
| 作为普通用户 | 我首次使用H5页面 | 需要能流畅打开，看到类似小程序的界面 |
| 作为用户 | 我需要登录 | 需要能通过手机号验证码方式登录 |
| 作为用户 | 我需要在多个页面间切换 | 需要底部TabBar支持页面切换 |

## 4. 技术约束

### 4.1 技术栈版本

| 技术 | 版本 | 说明 |
|------|------|------|
| Node.js | >= 18.0.0 | 项目运行环境 |
| Vue | 3.4+ | 核心框架 |
| Vite | 5.0+ | 构建工具 |
| TypeScript | 5.0+ | 类型支持 |
| Element Plus | 2.5+ | 后台UI组件库 |
| Vant | 4.x | H5移动端组件库 |
| Pinia | 2.1+ | 状态管理 |
| Vue Router | 4.2+ | 路由管理 |
| Axios | 1.6+ | HTTP请求 |

### 4.2 兼容要求

- Chrome 90+
- Safari 14+
- 移动端 Safari/Chrome
- 微信内置浏览器

---

## 5. 非功能需求

### 5.1 性能要求

- 首屏加载时间 < 3秒
- 路由切换时间 < 500ms
- 打包体积：后台 < 500KB (gzip)，H5 < 300KB (gzip)

### 5.2 代码规范

- ESLint + Prettier 配置
- Git Commit 规范（Conventional Commits）
- Vue 组件命名规范（大驼峰文件名，PascalCase 组件名）
